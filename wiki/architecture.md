# Architecture

> System design decisions for the Spotter ELD Trip Planner.

## Stack

| Layer | Choice | Reason |
|---|---|---|
| Backend | Django + DRF | Required by assessment |
| Frontend | React + Vite + TypeScript | Fast dev, Vercel-ready, strict type safety |
| Python pkg mgr | uv | Faster than pip, lockfile support |
| Map/Routing API | OpenRouteService | Free, no CC required, HGV profile |
| Log sheet render | SVG (React) | Precise pixel control, no canvas state mgmt |
| Backend deploy | DigitalOcean Droplet (BLR1) | $6/mo, 1GB RAM, Ubuntu 24.04 |
| Frontend deploy | Vercel | Free, zero config for Vite, auto CI/CD |
| Process manager | gunicorn (3 workers) + systemd | Production-grade, auto-restarts on reboot |
| Reverse proxy | nginx + Let's Encrypt SSL | HTTPS required (frontend is HTTPS, browser blocks mixed content) |
| CI/CD | GitHub Actions | Auto-deploys backend on push to `backend/**` |

## Deployment Architecture

```
Internet
   ↓
nginx (443 HTTPS) — api.asmittyagi.com
   ↓
gunicorn unix socket (/run/gunicorn.sock) — 3 workers
   ↓
Django app — /opt/logmate/backend
```

- **Frontend:** `logmate.asmittyagi.com` → Vercel (auto-deploys on push)
- **Backend:** `api.asmittyagi.com` → DigitalOcean 206.189.133.119 (deploys via GitHub Actions)
- **DNS:** Cloudflare — `api` A record DNS only (not proxied — required for Let's Encrypt)

## CI/CD

- **Frontend:** Vercel watches `main` branch, redeploys automatically on any push
- **Backend:** `.github/workflows/deploy-backend.yml` triggers only on `backend/**` changes — SSHes into server, git pulls, installs deps, restarts gunicorn

## Data Flow

```
User fills form (logmate.asmittyagi.com)
    ↓
React → POST /api/trip/ → Django (api.asmittyagi.com)
    ↓
Django geocodes 3 locations (ORS)
    ↓
Django fetches HGV route (ORS /v2/directions/driving-hgv/geojson)
    ↓
HOS calculator runs (pure, no network) — placeholder keys for stop locations
    ↓
Phase 2: parallel reverse geocode all stop positions (ThreadPoolExecutor)
    ↓
Returns JSON: { route, locations, days }
    ↓
React renders:
  - Step-based loading screen (4 steps, timer-driven)
  - Map with route polyline + C/P/D markers (Leaflet)
  - Trip summary card
  - ELD log sheets (SVG, paginated one day at a time)
```

### GET /api/autocomplete/?q=Chicago
Returns up to 5 location suggestions. Proxies ORS geocode/autocomplete with `layers=locality`.

Returns `{"status": "ok", "version": "1.0.0"}`. Used to verify backend is up.

## CORS

`CORS_ALLOWED_ORIGINS` in `settings.py` restricts browser requests to:
- `http://localhost:5173` (local dev)
- `http://127.0.0.1:5173` (local dev)
- `https://logmate.asmittyagi.com` (production)

CORS is browser-enforced only — direct `curl`/Postman calls are not blocked. Acceptable for V1.

## Error Handling

Custom exception hierarchy in `ors_client.py`:
- `LocationNotFoundError` → 422
- `RoutingError` → 422
- `RateLimitError` → 429
- `ServiceUnavailableError` → 503

Frontend `normalizeError()`: 422 passes through clean backend message, 400 reverts to generic (DRF field errors).

## TypeScript Config

Vite splits TypeScript into two configs:
- `tsconfig.json` — root, holds references only. `tsc --noEmit` alone checks **nothing**.
- `tsconfig.app.json` — real config with `strict: true`, `erasableSyntaxOnly`, `noUnusedLocals`.

**Rule:** always run `npm run typecheck` (= `tsc -b`). Never bare `tsc --noEmit`.

- `invalid_type_error` / `required_error` → replaced by unified `error` option
- `z.coerce.number()` has input type `unknown`, output type `number`
- Use `useForm<FormInput, unknown, FormOutput>` with `z.input<>` / `z.output<>` split

## HOS Calculator Architecture

Two-phase location resolution (avoids 10s sequential geocoding bottleneck):

```
Phase 1: HOS calculator runs with LocationCollector
  - No network calls
  - Each stop position stored as placeholder key ("__geo_0__", ...)

Phase 2: views.py resolves all positions in parallel
  - Deduplicate by rounded coordinate
  - ThreadPoolExecutor fires all reverse geocode calls simultaneously
  - Substitute real city names back into output
```

`RouteGeoRef` class in `hos_calculator.py` — haversine interpolation along ORS LineString geometry to get (lat, lng) at any cumulative mile position.

## Log Sheet Rendering

- Grid: 4 rows × 24hr, 15-min ticks, minor ticks at row boundaries
- Remarks: one diagonal per bracket (on-duty-not-driving period), not per status change
- Totals panel: HH:MM boxes, must sum to 24:00, on-duty decimal in red circle
- Navigation: one day at a time with prev/next chevrons

## V1 Scope

No auth, no saved trips, no mobile optimization. Core flow only: form → route → log sheets. Hosted and working.
